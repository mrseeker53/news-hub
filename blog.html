Arrow functions don't have their own "this", and they don't redefine the value of "this" within the function. This behavior of arrow functions makes them really useful when using callbacks inside methods.
In regular JavaScript functions, arguments keywords can be used to access the passed arguments when the function is invoked. But, arrow functions do not have their own arguments and it uses the arguments from the outer function.
However, arrow functions can not be used as constructors. If the arrow function contains one expression, you can omit the curly braces, and then the expression will be implicitly returned.
Regular functions are constructible, they can be called using the new keyword. However, arrow functions can never be used as constructor functions. Hence, they can never be invoked with the new keyword

Function ".forEach()," is used to execute the same code on every element in an array but does not change the array and it returns undefined. Function ".map()" executes the same code on every element in an array and returns a new array with the updated elements.
Functions".filter()" checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria.

Template literals (template strings) allow you to use strings or embedded expressions in the form of a string. They are enclosed in backticks ``. A multiline string: a string that can span multiple lines. String formatting: the ability to substitute part of the string for the values of variables or expressions. This feature is also called string interpolation.
HTML escaping: the ability to transform a string so that it is safe to include in HTML. A template tag carries the transformation on the template literal and returns the result string.
In this function: The literals parameter is an array that contains the literal strings. The substitutions parameter contains the subsequent arguments interpreted for each substitution.
